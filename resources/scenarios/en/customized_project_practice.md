## Base Development Guidelines

### Code Standards
- Follow the project's style guide
- Keep code concise, readable, and maintainable
- Use comments wisely to explain complex logic reasoning
- Avoid premature optimization, prioritize code clarity

### Naming Conventions
- Use meaningful names for variables and functions
- Follow project naming conventions (camelCase, snake_case, etc.)
- Use is/has/should prefixes for boolean variables
- Use UPPER_CASE with underscores for constants

### Error Handling
- Handle all operations that can fail
- Provide meaningful error messages
- Avoid empty catch blocks
- Use exceptions and error codes appropriately

### Security
- Validate all input data
- Prevent SQL injection, XSS, and other common vulnerabilities
- Never hardcode sensitive information
- Use environment variables for configuration

### Best Practices
- DRY principle: Don't Repeat Yourself
- SOLID principles: Single Responsibility, Open-Closed, etc.
- Keep functions and methods short with single responsibility
- Favor composition over inheritance
