## 前端开发场景规范

### UI 组件开发
- 所有组件必须支持响应式设计，适配不同设备
- 使用项目统一的设计系统和组件库
- 组件 props 必须有完整的 TypeScript 类型定义
- 复杂组件需要拆分为更小的子组件，保持单一职责
- 确保组件的可复用性和可测试性

### 样式规范
- 遵循项目 CSS 规范（CSS Modules/Tailwind/Styled-components）
- 支持深色模式（如果项目需要）
- 确保无障碍性（a11y），正确使用语义化 HTML
- 保持样式与整体设计系统一致
- 确保在不同浏览器和设备上显示正常

### 状态管理
- 使用项目约定的状态管理方案（Redux/Zustand/Context等）
- 避免 prop drilling，合理使用 Context 或状态库
- 异步状态必须处理 loading 和 error 状态
- 状态结构设计要清晰，避免冗余
- 合理使用本地状态和全局状态

### 性能优化
- 合理使用 React.memo、useMemo、useCallback
- 避免不必要的重渲染
- 大列表使用虚拟滚动（react-window/react-virtualized）
- 图片懒加载和优化
- 代码分割和按需加载

### 测试要求
- 每个组件都要有对应测试文件
- 测试用户交互和边界情况
- 使用 Testing Library 测试用户行为而非实现细节
- 保持测试简洁和可维护

### 开发注意事项
- 避免内联函数和对象（除非性能影响可忽略）
- 正确处理表单验证和提交
- 合理使用 useEffect，避免依赖问题
- 注意内存泄漏，及时清理定时器和订阅
