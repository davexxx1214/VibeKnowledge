## 后端开发场景规范

### API 实现
- RESTful API 遵循标准的 HTTP 方法语义
- 统一的响应格式和错误码定义
- 合理的状态码使用（2xx/4xx/5xx）
- API 版本控制和向后兼容性
- 完善的 API 文档（Swagger/OpenAPI）

### 数据库设计
- 合理的表结构设计，遵循范式
- 正确设置索引，优化查询性能
- 外键约束和数据完整性
- 数据库迁移脚本版本管理
- 考虑数据扩展性和分区策略

### 中间件开发
- 统一的错误处理中间件
- 请求日志和性能监控
- 认证和授权中间件
- 请求验证和参数过滤
- 限流和防护措施

### 服务层设计
- 业务逻辑与控制器分离
- 服务的单一职责原则
- 合理使用依赖注入
- 事务管理和数据一致性
- 缓存策略（Redis/Memcached）

### 第三方服务集成
- SDK 配置和初始化
- API 调用封装和错误处理
- 重试机制和熔断器
- 超时控制和资源管理
- 完整的单元测试和集成测试

### 安全性
- 输入验证和参数过滤
- SQL 注入防护
- XSS/CSRF 防护
- 敏感数据加密存储
- API 访问控制和权限管理

### 性能优化
- 数据库查询优化，避免 N+1 问题
- 合理使用缓存
- 异步处理和消息队列
- 连接池和资源复用
- 监控和性能分析
