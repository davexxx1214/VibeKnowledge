## 文档编写场景规范

### API 文档

#### 接口列表
- 完整的端点列表和分组
- HTTP 方法和路径
- 简要功能描述
- 认证要求说明

#### 请求参数
- 参数名称和类型
- 是否必填
- 默认值
- 取值范围和格式
- 示例值

#### 响应格式
- 成功响应示例（JSON）
- 字段说明和类型
- 可能的响应状态码
- 分页信息（如适用）

#### 错误说明
- 错误码列表
- 错误信息描述
- 错误场景说明
- 错误响应示例

#### 请求示例
- curl 命令示例
- 不同编程语言的调用示例
- 认证 Token 使用方法
- 完整的请求响应流程

### 代码注释

#### 函数/方法注释
```typescript
/**
 * 计算两个数的和
 * 
 * @param a - 第一个加数
 * @param b - 第二个加数
 * @returns 两数之和
 * @throws {TypeError} 如果参数不是数字
 * @example
 * ```ts
 * const result = add(1, 2); // 3
 * ```
 */
function add(a: number, b: number): number {
  return a + b;
}
```

#### 类注释
- 类的职责和用途
- 主要属性说明
- 使用示例
- 注意事项

#### 复杂逻辑注释
- 解释"为什么"而不是"是什么"
- 算法思路和边界情况
- 性能考虑
- 已知限制

- 项目名称和描述
- 主要功能特性
- 技术栈
- 项目状态（开发中/稳定/维护）

#### 快速开始
- 环境要求
- 安装步骤
- 配置说明
- 运行方法
- 示例命令

#### 项目结构
- 目录说明
- 主要文件作用
- 模块划分

#### 开发指南
- 开发环境配置
- 代码规范
- 提交规范
- 测试方法
- 调试技巧

#### 部署说明
- 构建命令
- 环境变量配置
- 部署步骤
- 常见问题

#### 贡献指南
- 如何贡献代码
- PR 提交流程
- Code Review 流程
- Issue 提交规范

### 技术文档

#### 架构文档
- 系统架构图
- 模块划分和职责
- 技术选型理由
- 数据流向

#### 设计文档
- 功能需求
- 技术方案
- 接口设计
- 数据库设计
- 风险和限制

#### 变更日志（CHANGELOG）
- 版本号和发布日期
- 新功能
- 修复的问题
- 破坏性变更
- 弃用说明

### 文档最佳实践
- 保持文档与代码同步
- 使用清晰的语言，避免歧义
- 提供完整的示例
- 使用图表辅助说明
- 定期审查和更新文档
- 考虑不同读者的需求（用户/开发者）
